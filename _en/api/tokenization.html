The Tokenization product allows you to store your customers’ credit card data safely, through the creation of a token, with which you can make regular charges or implement 1 Click payment functionality, thus following PCI DSS (Payment Card Industry Data Security Standard)security standards handling credit card data.
In order to use the product’s tokenization functionalities from the API, you must have this service enabled on your PayU account, if you want to purchase it contact sac@payulatam.com.  
 
How does it work?  
The "Tokenization" product includes two separate processes:

 
 
Available methods for tokenization are: Individual credit card registration, Massive credit card registration, Payment with token, Massive payments with token, Token query and Token removal. Below is a description of each.

Massive credit card registration
 
Using this feature you can register various customer’s credit card data and get token sequential numbers.
For massive credit card tokenization you must create a file with CSV format with the following conditions:  
  1. Each credit card register to be tokenized must contain data for the registration of a credit card, separated by commas in the following order: Payer ID, full name, credit card number, expiration date, franchise, and identification number.
  2. The file should have no header. The first line shows the first record.
  3. The file must be encoded under the UTF-8 standard.
  4. The file must not have more than 10,000 registers.
 

  • JSON
  • XML
Request example: 
{
   "language": "es",
   "command": "CREATE_BATCH_TOKENS",
   "merchant": {
      "apiLogin": "11959c415b33d0c",
      "apiKey": "6u39nqhq8ftd0hlvnjfs66eh8c"
   },
   "contentFile": "MDAxLG5vbWJyZSBhcGVsbGlkbyAxLDQxMTExMTExMTExMTExMTEsMjAxNS8wMSxWSVNBLDEyMzQ1Ng=="
}
 
Response example: 
{
   "code": "SUCCESS",
   "error": null,
   "id": "edd187bb-067c-4016-b94d-3a4727cc86eb"
}
 
You must implement a function that takes the contents of the file and pass it to base 64 and send this file in the parameter <contentFile>.

Massive payments with token
 
This feature will allow you to make massive collections using the Token that were previously created by our system
For this functionality you need to create a CSV file under the following conditions:  
  1. Each transaction register to be processed must contain its processing data separated by commas in the following order: Account ID (identifier of the virtual PayU account), Token, security code, number of installments, sale description, buyer’s email, currency (ISO code), total (including tax), tax, base value of reimbursement, additional value, language (ISO code).
  2. The file should have no header. The first line shows the first record.
  3. The file must be encoded under the UTF-8 standard.
  4. The file must not have more than 10,000 records.
 

  • JSON
  • XML
Request example: 
{
   "language": "es",
   "command": "PROCESS_BATCH_TRANSACTIONS_TOKEN",
   "merchant": {
      "apiLogin": "11959c415b33d0c",
      "apiKey": "6u39nqhq8ftd0hlvnjfs66eh8c"
   },
   "contentFile": "NTAwNTM4LGVhMDIwZTU5LWQ5NWEtNDk1ZC04OTAzLTM0ZTg0M2ZkN2ZlYywxMzIsMSxwcnVlYmEtMDQscHJ1ZWJhLHNlYmFzdGlhbi5xdWludGVyb0BwYWdvc29ubGluZS5jb20sQ09QLDEwMDAwLDAsMCwwLGVz=="
}
 
Response example: 
{
   "code": "SUCCESS",
   "error": null,
   "id": "b85cfd29-4765-4502-98d7-5fd8c8b4cc84"
}
