API Promotions in Argentina This function will allow you to consult via API the valid promotions, their associate costs, characteristics and further relevant information that you may show to your clients.  
Authentication
We suggest to use a HMAC based mechanism to authenticate and authorize the petitions received by your server. In order to make the authentication, you need to know your MerchantPublicKey whose information may be found in the tab "Settings", option "technical information" under the name "Public Key".

The header should have the following structure:  
Authorization Header: "Hmac" + " " + MerchantPublicKey + ":" + Signature"

Where the Signature is created as follows:  
Signature = Base64(HMAC-SHA256(MerchantApiKey,ContentToSign))

The ContentToSign corresponds to:  
HTTP-Verb + "\n" + "\n" + "\n" +
Date + "\n" +
URI

Note:
  1. It is necessary to leave 3 spaces in the example line!

Next, you may see an example with the following test data: 
ContentToSign:
GET 
 

Tue, 05 Ago 2015 18:32:01 GMT
/payments-api/rest/v4.3/pricing

MerchantApiKey: 6u39nqhq8ftd0hlvnjfs66eh8c
MerchantPublicKey: PK18e9t2p6k56Xe31o5jrv4Nft
The above information is related and the HMAC function is applied as it follows: 
Authorization: Hmac PK18e9t2p6k56Xe31o5jrv4Nft:QCtnCLaYGpI7djR22o9aqBAzL3ysaS1xZMTv0HaX+N0=

To avoid Replay attacks, you have to send the header "Date". Because of the restrictions seen in some REST clients, you may also send the header "x-hmac-date" to comply with the security requirements, as shown in this example: 
Date =	 Mon, 11 May 2015 21:14:41 GMT
In the event that the REST client has any restriction you may send: 
x-hmac-date = Mon, 11 May 2015 21:14:41 GMT
Next, we will describe the attributes and types of fields for each variable returned from PayU in case of consulting the API of Pricing and Promotions.

Pricing model variables

Ownership Lenght Type Description
paymentMethodFeeDetail.paymentMethod String Name of means of payment. Possible values MASTERCARD, VISA, AMEX, etc.
fee Fee Object that has all the pricing for a means of payment and installment, or all the installment (for countries that have a installment-based system).
fee.installments String Installment (1) or set of installments (1-36 for countries with installment-based ranges).
fee.pricing Pricing It has the Pricing values by default for the transaction.
fee.pricing.payerDetail PayerDetail It has the values of interests and fees that will be applied to the payer.
fee.pricing.payerDetail.commission Decimal Total fees to be applied to the payer, includes fees and taxes, if applicable.
fee.pricing.payerDetail.interests Decimal Total interests to be applied to the payer, includes fees and taxes, if applicable.
fee.pricing.payerDetail.total Decimal The total value to be paid by the payer for fees and interests.
fee.pricing.merchantDetail.commission Merchant Detail Total fees to be applied to the business, includes fees and taxes, if applicable.
fee.pricing.merchantDetail.interests Decimal Total interests to be applied to the business, includes fees and taxes, if applicable.
fee.pricing.merchantDetail.total Decimal The total value to be paid by the business because of fees and interests.
fee.pricing.totalValue Decimal Payment total, is the value submitted by the business plus the fees and interests that may correspond to the payer.
amount.value Decimal Corresponds to the value sent by the business with the purchase value, it may be in a currency other than that of the account.
amount.tax Decimal Corresponds to the value sent by the business with the tax value, it may be in a currency other than that of the account.
amount.taxableBase Decimal Corresponds to the value sent by the business with the taxable base, it may be in a currency other than that of the account. This value is normally used in Colombia.
amount.currency 3 String It is the currency that the business sends the transaction with (COP, USR, BRL, ARS).
convertedAmount Converted Amount The incoming amounts are related converted into the local currency. These values are shown only os the currency sent is different to the currency of the country.
convertedAmount.value Decimal Amount of the transaction, in the currency used by the account.
cconvertedAmount.tax Decimal Tax value.
convertedAmount.taxableBase Decimal Taxable base. In Colombia this is used a tax refund.
convertedAmount.currency 3 String It is the currency corresponding to the account sent (COP, USR, BRL, ARS)

Promotions model variables

Ownership Lenght Type Description
promotion.id Integer Identifier of promotion in PayU platform.
promotion.title 50 String Promotion title.
promotion.termsAndConditions 250 String Description of the terms and promotions that apply to the promotion.
promotion.paymentMethod String Name of means of payments applied to the promotion. Possible values MASTERCARD, VISA, AMEX, etc.
promotion.subFranchise String Name of sub-brand/sub-franchise the promotion is made for. Some sub-brands/sub-franchises are: Nativa, Naranja, Cencosud.
promotion.banks List List of banks where the promotion is valid.
promotion.iins List List with IIN/BINES of the banks, that the promotion applies to.
promotion.days List List of days where the promotion is valid.
promotion.startDate Datetime Date when promotion starts.
promotion.endDate Datetime Date when promotion ends.
