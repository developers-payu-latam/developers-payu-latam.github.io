API integration - queries

{% include sidemenu_api_en.html %}
Queries with api PayU provides an API that allows you to check the status of transactions that are generated from placed orders. If you want to integrate with our API you must target the requests to the following URLs:
  • API
Tests: https://sandbox.api.payulatam.com/reports-api/4.0/service.cgi
Production: https://api.payulatam.com/reports-api/4.0/service.cgi
Consider:
  • You must use the API to query when transactions remain in pending status or PENDING.
  • You must perform the first query 7 minutes after the transaction was created.
  • Please don’t retry the query of an order or transaction if the system indicates that it does not exist.
  • For cash payment methods, you can perform only one query per hour.
  • Transactions with payment means such as credit card or bank transfer are available only once every 10 minutes
The methods available for the query API are: PING, Query order by identifier, Query order by reference and Transaction response query. Below is a description of each.
PING
You can use the doPing method to verify connectivity to our platform.
  • JSON
  • XML
Request example: 
{
   "test": false,
   "language": "en",
   "command": "PING",
   "merchant": {
      "apiLogin": "pRRXKOl8ikMmt9u",
      "apiKey": "4Vj8eK4rloUd272L48hsrarnUA"
   }
}
 
Response example: 
{
   "code": "SUCCESS",
   "error": null,
   "transactionResponse": null
}
Request example: 
<request>
   <language>en</language>
   <command>PING</command>
   <merchant>
      <apiLogin>pRRXKOl8ikMmt9u</apiLogin>
      <apiKey>4Vj8eK4rloUd272L48hsrarnUA</apiKey>
   </merchant>
   <isTest>false</isTest>
</request>
 
Response example: 
<paymentResponse>
   <code>SUCCESS</code>
</paymentResponse>
