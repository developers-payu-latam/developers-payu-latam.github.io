To consider  
If you want to make tests through PayU, since the test environment does not replicate data from your account in production, it is important that you use the following credentials in the request, depending on the country of your account:  
{% include test_credentials_en.html %}  
 
Aiming at the following URLs:
Queries: https://stg.api.payulatam.com/reports-api/4.0/service.cgi

Payments: https://stg.api.payulatam.com/payments-api/4.0/service.cgi
 
Note:
  1. It is important that during the performance of tests you do not use real Credit Card data, you can use Credit Card generators.
  2. Tests may only be done using Credit Cards, payment means such as cash or bank transfer cannot be tested by this method.
  3. Payment means in Peru cannot be tested, since these franchises are handled through 3D security.
 
 
 
Testing statuses  
When using the Authorization or Authorization and capture request you must:
  1. Ensure sending the test parameter as true.
  2. For the cardholder value enter the name APPROVED if you want the transaction to be approved, REJECTED if you want it to be rejected or PENDING if you want it to be pending.
  3. For the card number you must enter a valid card number, corresponding to the franchise sent in the request.

Stepping over to production  
After you make all system testing and you have verified that everything is working properly, you can change to the production server; where the actual transactions are processed.  
For this you should uncheck the option transaction in test mode, which you'll find in your administrative module.  

 
Additionally you must create payment forms with the following parameters corresponding to your account:  
merchantId: from your account
ApiKey: from your account
ApiLogin: from your account
accountId: from your account
signature: the signature is generated for each transaction
test: false
 
Pointing to production URLs:
Queries: https://api.payulatam.com/reports-api/4.0/service.cgi

Payments: https://api.payulatam.com/payments-api/4.0/service.cgi
